# Quality Standards

## Accuracy
- [ ] Every command is real and verified.
- [ ] Every config property exists in the codebase.
- [ ] Outputs are from actual execution logs.

## Clarity
- [ ] Written for users encountering the feature for the first time.
- [ ] Context and setup steps are included.
- [ ] Explanations are clear and concise.

## Completeness
- [ ] Includes setup commands.
- [ ] Includes exact CLI commands.
- [ ] Includes expected results.

## Consistency
- [ ] Matches the style of existing case studies.
- [ ] Uses proper markdown formatting.

## Practicality
- [ ] Focuses on realistic scenarios.
- [ ] Uses cost-effective models where possible.
